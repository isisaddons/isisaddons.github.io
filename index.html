Forking the Repos

All repos follow a standard structure. In the case of the isis-module-xxx add-ons, these have the following structure:

  • dom - domain object module
  • fixture - fixtures
  • integtests - integration tests
  • webapp - webapp to run

In the case of the isis-wicket-xxx add-ons, the structure is:

  • cpt - component module
  • fixture - fixtures
  • webapp - webapp to run, and webapp (UI) tests

Only the dom (for module add-ons) or cpt (for wicket add-ons)is released into Maven central repo; this is all that is required if you have no need to customize the functionality.

If you do need to modify/adapt the provided functionality, then fork the entire repo; this way you also have fixtures and tests to ensure that your changes are properly tested. And, if you think others would benefit from your changes, please consider raising a pull request to fold those changes back into the original.

Other Add-ons:
