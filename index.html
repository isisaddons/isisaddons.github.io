This site catalogs a number of add-ons to Apache Isis for you to use either "out of the box", or as a starting point to modify and adapt to your own requirements. All add-ons live in the org.isisaddons package.

Each add-on is hosted in its own github repo, named as follows:

  • isis-module-xxx : a domain object "module" that implements business logic for a particular subdomain, eg application settings, or tags (labels).
  • isis-wicket-yyy : an extension to Isis' Wicket viewer, eg for rendering calendars or maps.

There are also a number of other add-ons that either do not fit into the above categories, or which do not follow the above package requirements, but which may nevertheless be of interest to developers building apps with Apache Isis. The repos for these additional add-ons have been forked into this site for convenience.

Structure

All repos follow a standard structure. In the case of the isis-module-xxx add-ons, these have the following structure:

  • dom - domain object module (released)
  • fixture - fixtures
  • integtests - integration tests
  • webapp - webapp to run

In the case of the isis-wicket-xxx add-ons, the structure is:

  • lib - library module (released)
  • fixture - fixtures
  • webapp - webapp to run, and webapp (UI) tests

Only the dom (for module add-ons) or lib (for wicket add-ons)is released into Maven central repo; this is all that is required if you have no need to customize the functionality.

If you do need to modify/adapt the provided functionality, then fork the entire repo; this way you also have fixtures and tests to ensure that your changes are properly tested. And, if you think others would benefit from your changes, please consider raising a pull request to fold those changes back into the original.

READMEs

Each add-on has its own README; use that to learn more about the features provided by the add-on and how to install/configure the add-on. Some add-ons also have wiki pages (linked from the README).
