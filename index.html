    Repo Structure

    The isis-module-xxx add-ons have the following structure:

    • dom - domain object module
    • fixture - fixtures
    • integtests - integration tests
    • webapp - webapp to run

    The isis-wicket-xxx add-ons have the structure:

    • cpt - component module
    • fixture - fixtures
    • webapp - webapp to run, and webapp (UI) tests

    Use "Out-of-the-box"

    Only the dom (for module add-ons) or cpt (for wicket add-ons) is released into Maven central repo; this is all that is required if you have no need to customize the functionality.

    Forking the Repos

    Alternatively, if you do need to modify/adapt the provided functionality, then fork the entire repo. This way you also have fixtures and tests to ensure that your changes are properly tested.

    And, if you think others would benefit from your changes, please consider raising a pull request to fold those changes back into the original.

    Other Add-ons:

    There are also a number of other add-ons that either do not fit into the above categories, or which do not follow the above package requirements, but which may nevertheless be of interest to developers building apps with Apache Isis. The repos for these additional add-ons have been forked into this site for convenience.
