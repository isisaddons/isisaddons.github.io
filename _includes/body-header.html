Each add-on is hosted in its own github repo:

  • isis-module-xxx : a domain object "module" that implements business logic for a particular subdomain, eg application settings, or tags (labels).
  • isis-wicket-yyy : a component for Isis' Wicket viewer, eg for rendering calendars or maps.

READMEs

Each add-on has its own README; use that to learn more about the features provided by the add-on and how to install/configure the add-on. Some add-ons also have wiki pages (linked from the README).

Forking the Repos

All repos follow a standard structure, including a demo webapp and tests, to make them easy to fork and extend.

Other Add-ons

There are also a number of other add-ons that either do not fit into the above categories, or which do not follow the above package requirements, but which may nevertheless be of interest to developers building apps with Apache Isis. The repos for these additional add-ons have been forked into this site for convenience.
